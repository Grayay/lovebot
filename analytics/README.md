# Сравнение выдачи с генеративными технологиями и без через тематическое моделирование

Скрипт `topic_modeling.py` сравнивает тексты страниц, которые получили пользователи:
- из варианта с генеративными ответами (`generative`)
- из варианта без генеративных ответов (`baseline`)

Используется LDA (Latent Dirichlet Allocation) и считается расхождение распределений тем через **Jensen-Shannon divergence**.

## Входные данные
Поддерживаются форматы `CSV` и `JSONL` с обязательными полями:
- `query` — текст запроса или его ID
- `variant` — тип выдачи (`gen`, `generative`, `ai`, `baseline`, `organic`, `classic` и т.д.)
- `text` — текст страницы, показанной пользователю

Пример CSV:

```csv
query,variant,text
"лучшие кредитные карты",generative,"..."
"лучшие кредитные карты",baseline,"..."
```

## Запуск

```bash
python analytics/topic_modeling.py data/serp_texts.csv --topics 8 --top-words 12 --output reports/topic_report.md
```

## Интерпретация
- **Jensen-Shannon divergence** ближе к `0` → тематическая структура похожа.
- Чем выше значение (максимум около `1` при log2), тем сильнее различия между наборами выдачи.
- В отчете также будет:
  - список ключевых слов для каждой темы;
  - средняя доля каждой темы в `generative` и `baseline`.

## Практический пайплайн
1. Снять тексты страниц из логов/краулера/аналитики SERP.
2. Сформировать единый датасет с `query`, `variant`, `text`.
3. Запустить скрипт и получить markdown-отчет.
4. Проверить темы с наибольшей разницей долей для продуктовых выводов.
